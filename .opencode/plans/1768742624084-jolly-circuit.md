# Plan: Danube monorepo layout

## Goals
- Establish a monorepo layout with a root Python folder(backend) and a root TypeScript(frontend) folder, plus infra.
- Centralize configuration in root-level `pyproject.toml` and `package.json` as requested.
- Include `justfile` for repo-level orchestration and basic root scaffolding files.

## Recommended layout
```
backend/
  master/
  coordinator/
  worker/
  api/
frontend/
infra/
  helm/
docs/                (existing)
examples/
tests/
scripts/
tools/
```

### Rationale
- `backend/` is the single top-level backend area, with separate service folders that align to Master/Coordinator/Worker roles.
- `frontend/` is the top-level UI/TS area
- `infra/helm/` is the initial infra scope, per your preference.
- `docs/` stays untouched; documentation updates can follow once the layout is in place.

## Root-level tooling strategy
- Root-level `pyproject.toml` and `package.json` provide the canonical tool configuration and package metadata for the repo.
- Keep configs flat (no workspaces); subfolders are standard modules/projects referenced by tooling or scripts.

## Files to add (initial scaffolding)
- `README.md`: high-level overview, how to run each subproject, pointers to docs.
- `CONTRIBUTING.md`: setup steps for backend/frontend/infra with UV/Node versions.
- `.editorconfig`: enforce consistent formatting across the repo.
- `justfile`: common commands delegating to subprojects (e.g., `just backend-test`, `just frontend-dev`).
- `.gitignore`: repo-wide ignores.

## Implementation steps
1. Create top-level directories: `backend/`, `frontend/`, `infra/helm/`, `docs/` (existing), `examples/`, `tests/`, `scripts/`, `tools/`.
2. Add backend component folders under `backend/`: `master/`, `coordinator/`, `worker/`, `api/`.
3. Add root scaffolding files (`README.md`, `CONTRIBUTING.md`, `.editorconfig`, `justfile`, `.gitignore`).
4. Add root config files (`pyproject.toml`, `package.json`).
5. Update documentation references only if needed once implementation begins (optional follow-up).

## Verification
- No runtime tests required for layout-only change.
- Optional sanity check: `just --list` once `justfile` exists.
